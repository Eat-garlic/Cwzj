根据您提供的信息，这是一个针对农信银行外汇业务数据报送到外汇管理局的系统。下面我会按照您的描述来概述该系统的主要功能、用户角色和技术特点。

### 主要功能

1. 管理员功能
   - **报送员管理**：管理员可以添加、删除、修改和查询报送员的账户信息。
   - **权限管理**：管理员可以分配和修改报送员的权限，确保每个报送员只能访问和操作其被授权的数据和功能。
2. 报送员功能
   - **报送数据维护**：报送员可以录入、修改和删除外汇业务数据。
   - **报送数据生成并上报**：报送员可以将维护好的数据生成特定的格式（如XML、JSON等），并将其打包成ZIP文件，通过系统上报到外汇管理局。
   - **反馈数据解析**：系统接收外汇管理局返回的反馈数据后，报送员可以解析这些数据，查看报送的结果和可能的错误。

### 责任描述

1. **负责报送数据的CRUD**：系统需要支持对外汇业务数据的增（Create）、读（Read）、更（Update）和删（Delete）操作。
2. **报送数据zip文件的生成并上报**：系统需要能够自动或手动地将报送数据打包成ZIP格式，并通过安全的通道（如HTTPS）上报到外汇管理局。
3. **反馈数据的拉去和解析**：系统需要能够从外汇管理局拉取反馈数据，并自动解析这些数据，将其转换为报送员可以理解的格式。

### 技术描述

1. **系统集群部署**：为了提高系统的可用性和性能，系统采用集群部署的方式，即多个服务器同时运行系统，并通过负载均衡技术分配用户的请求。
2. **使用nginx做负载均衡**：nginx是一个高性能的HTTP和反向代理服务器，它可以有效地将用户的请求分发到不同的服务器，确保每个服务器都能均衡地处理请求，从而提高系统的整体性能。

### 总结

这个农信银行外汇业务数据报送系统是一个复杂且专业的系统，它要求管理员和报送员能够高效、准确地管理和报送外汇业务数据，同时还需要确保系统的稳定性和安全性。通过集群部署和使用nginx进行负载均衡，系统可以满足高性能和高可用性的要求，从而满足银行业务的需求。

# git bash报错fatal: detected dubious ownership in repository at的解决方法









原因
文件夹的所有者和现在的用户不一致

例如：文件夹的所有者是Administrator，而当前用户是myAccount

解决方案一
右键文件夹 -> 属性 -> 所有者 -> 更改 -> 勾选应用到所有子目录和文件 -> 应用

解决方案二（简便）
在git bash中输入

git config --global --add safe.directory "*"
